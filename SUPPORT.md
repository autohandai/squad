# Support

Start with the [README](README.md) and [setup guide](SETUP_GUIDE.md). They cover
installation, development modes, local state, validation, and common runtime
problems.

If those guides do not answer a setup or usage question, use the
[question form](https://github.com/autohandai/squad/issues/new?template=question.yml)
with the relevant version, environment, commands, and secret-free output.

For a reproducible bug, use the
[bug report form](https://github.com/autohandai/squad/issues/new?template=bug_report.yml).
Search existing issues first and include the Squad version, operating system,
steps to reproduce, expected result, and secret-free logs.

For a proposed feature or workflow change, use the
[feature request form](https://github.com/autohandai/squad/issues/new?template=feature_request.yml)
to describe the user problem before the implementation. Keep large design and
architecture proposals small enough to discuss and review incrementally.

- Contribution help: [CONTRIBUTING.md](CONTRIBUTING.md)
- Security vulnerabilities: [SECURITY.md](SECURITY.md)
- Community conduct concerns: [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md)

This is an open-source project, so response times can vary. Please do not post
the same request in several issues or include credentials, private prompts,
customer data, or local Autohand configuration.
